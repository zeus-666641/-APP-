# APP制作骨架（核心索引）

> 任何时候操作都必须对照此骨架。每个项目（环节）→流程→步骤→Skill映射。
> 详细步骤见 `完整制作流程清单.md`。

## 7大项目总览

| # | 项目 | 目标 | Skill | 关键产出 |
|---|---|---|---|---|
| 1 | 选题与立项 | 决定做什么 | app-ideation | `docs/prd/draft.md` |
| 2 | 产品规划 | 转成开发计划 | (待建)prd-generator | PRD+架构+任务清单 |
| 3 | 架构与脚手架 | 生成项目骨架 | flet-scaffold | `apps/{app}/` 可运行骨架 |
| 4 | 核心编码 | 实现所有功能 | flet-scaffold + Subagent | 完整代码+测试 |
| 5 | 质量保障 | 质量达标 | (待建)flet-test-runner、mobile-perf-auditor | 测试报告、优化代码 |
| 6 | 打包封装 | 生成签名APK | flet-cloud-builder | `releases/{app}_{ver}.apk` |
| 7 | 发布与运营 | 上架+迭代 | (待建)play-store-publisher | 商店版本+运营数据 |

## 项目1：选题与立项（3流程/11步骤）

### 流程1.1 市场调研
- 1.1.1 关键词扩展 → WebSearch
- 1.1.2 竞品收集 → WebSearch+应用商店
- 1.1.3 用户痛点挖掘 → WebSearch
- 1.1.4 趋势判断 → WebSearch

- 1.2.1 候选方案设计（3个：优化型/细分型/创新型）
- 1.2.2 MVP范围界定（3-5个核心功能）

### 流程1.3 立项决策
- 1.3.1 用户选择 → AskUserQuestion
- 1.3.2 方案归档 → Write到`记忆/决策.md` + `docs/prd/draft.md`

**产出**：`docs/prd/draft.md`、`记忆/选题调研_{时间}.json`

## 项目4：核心编码（6流程/24步骤）⭐工作量最大

### 流程4.1 数据层
- 4.1.1 数据模型 → `models/{entity}.py`
- 4.1.2 存储层 → `services/storage.py`
- 4.1.3 数据迁移 → `services/migrations.py`
- 4.1.4 单测 → `tests/test_models/`

### 流程4.2 业务层
- 4.2.1 Service类 → `services/{domain}.py`
- 4.2.2 异步处理（async/await）
- 4.2.3 错误处理（AppError基类）
- 4.2.4 单测 → `tests/test_services/`

### 流程4.3 UI层
- 4.3.1 页面View → `views/{page}_view.py`
- 4.3.2 可复用组件 → `views/components/`
- 4.3.3 路由注册 → `routes.py`
- 4.3.4 状态管理
- 4.3.5 UI测试 → `tests/test_views/`

### 流程4.4 网络层（如需）
- 4.4.1 API客户端（httpx）
- 4.4.2 请求拦截
- 4.4.3 缓存策略
- 4.4.4 单测（respx mock）

### 流程4.5 权限与原生功能
- 4.5.1 权限声明（AndroidManifest.xml）
- 4.5.2 权限申请（Flet permission API）
- 4.5.3 原生调用（相机/位置/通知）

### 流程4.6 国际化
- 4.6.1 文案提取 → `i18n/zh.json`、`en.json`
- 4.6.2 翻译函数 → `services/i18n.py`
- 4.6.3 语言切换

**产出**：完整可运行App代码 + 测试套件

## 操作准则（每次执行前必读）

1. **对照骨架定位**：明确当前在哪个项目/流程/步骤
2. **检查前置条件**：上游产出物是否就绪
3. **按步骤执行**：不跳步、不合并
4. **写入产出物**：每步完成立即归档
5. **更新日志**：在 `操作日志.md` 追加记录
6. **失败暂停**：任何失败不静默，写入日志后暂停
