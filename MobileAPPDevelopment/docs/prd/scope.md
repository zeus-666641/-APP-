# 步骤记录器 · Flet 实现范围清单

> 基于 `步骤记录器v1.0` PRD × Flet 0.86 能力对照
> 用户决策：按 Flet 子集实现，明确告知不可实现项，分模块逐步推进，质量优先

## 评估方法

每项功能按以下三档标注：
- ✅ **可做**：Flet 直接支持，无障碍实现
- ⚠️ **受限做**：Flet 部分支持，能做但功能受限或需变通
- ❌ **不可做**：Flet 完全无 API，依赖 Android 原生（AccessibilityService / AlarmManager / WindowManager 等）

## 二、步骤类型逐项评估（PRD 第 5 章）

### 5.1 模拟操作类

| 步骤 | 评估 | 说明 / 变通 |
|---|---|---|
| 模拟点击 | ⚠️ | App 内控件点击 ✅；跨应用点击 ❌（需 AccessibilityService） |
| 模拟滑动 | ⚠️ | App 内控件滑动 ✅；跨应用 ❌ |
| 返回主界面 | ⚠️ | App 内 NavigationBar 切换 ✅；系统 Home 键 ❌ |
| 返回上一级 | ⚠️ | App 内 `page.pop()` ✅；系统返回键 ❌ |
| 进入应用 | ❌ | 跨应用启动，Flet 无法 |
| 切换应用 | ❌ | 同上 |
| 拨号按键 | ⚠️ | Flet 可启动拨号 intent，但无法自动拨打 |
| 打开链接 | ✅ | `page.launch_url()` 直接支持 |

### 5.2 系统控制类（**全部不可做**）

| 步骤 | 评估 | 原因 |
|---|---|---|
| 蓝牙 开/关 | ❌ | 需 BLUETOOTH_ADMIN + 系统签名 |
| WiFi 开/关 | ❌ | 需 CHANGE_WIFI_STATE + 系统 API |
| 飞行模式 开/关 | ❌ | 系统签名权限 |
| 定位 开/关 | ❌ | 需 LOCATION 权限 + 系统 API |
| 个人热点 开/关 | ❌ | 系统签名 |
| NFC 开/关 | ❌ | 仅系统设置页跳转，无 API |
| 移动数据 开/关 | ❌ | 系统签名 |
| 勿扰模式 开/关 | ❌ | 需 NOTIFICATION_POLICY_ACCESS |
| 省电模式 开/关 | ❌ | 系统签名 |
| 专注模式 开/关 | ❌ | 系统 API |
| 驾车模式 开/关 | ❌ | 系统 API |
| 锁屏 | ❌ | 需 Device Admin |
| 控制屏幕息屏 | ❌ | 系统签名 |
| 开关机 | ❌ | 需 root |

### 5.3 显示设置类

| 步骤 | 评估 | 说明 |
|---|---|---|
| 调节屏幕亮度 | ❌ | 系统设置，需 WRITE_SETTINGS |
| 方向锁定 | ❌ | 系统 API |
| 开关深色模式 | ✅ | App 内主题切换（非系统级） |
| 调整显示大小 | ❌ | 系统 API |
| 设置字体大小 | ❌ | 系统 API |
| 设置字体粗细 | ❌ | 系统 API |
| 开关护眼模式 | ❌ | 系统签名 |
| 屏幕常亮(开) | ✅ | `page.window_always_on_top` 或 keep_alive |
| 屏幕常亮(关) | ✅ | 同上 |

### 5.4 音频与触感类

| 步骤 | 评估 | 说明 |
|---|---|---|
| 调节音量 | ❌ | AudioManager，Flet 无 |
| 设置静音 | ❌ | 同上 |
| 改变系统触感 | ❌ | 系统 API |
| 设备振动 | ⚠️ | Flet 0.86 有 `page.vibration(duration)`，可做但限制大 |

### 5.5 辅助与通知类

| 步骤 | 评估 | 说明 |
|---|---|---|
| 发送通知 | ⚠️ | Flet 有 `ft.Notification`，可发本地通知（需 Android 通知通道） |
| 弹出提醒 | ✅ | `ft.AlertDialog` 直接支持 |
| 延时等待 | ✅ | `asyncio.sleep()` |

| 开启手电筒 | ❌ | Camera2 API，Flet 无 |

## 六、能力总览

- **完全可做**：约 40%（任务管理 UI、步骤编辑器 UI、数据模型、统计、日志、UI 规范、深色模式、弹出提醒、延时等待、子任务、打开链接、屏幕常亮、深色模式切换）
- **受限可做**：约 15%（前台定时器、本地通知、震动、App 内导航/点击/滑动、拨号 intent 启动）
- **完全不可做**：约 45%（系统控制 14 项 + 模拟跨应用操作 2 项 + 屏幕亮度/字体/护眼 + 音量/静音/触感 + 屏幕OCR + 悬浮窗 + 后台触发器 + 后台条件监听 + 手电筒）

---

## 七、待用户决策的处理方式

对每一类"不可做"项，有 4 种处理方式：
1. **完全删除**：从 PRD 中删除，不展示给用户
2. **置灰显示**：在 UI 中显示但禁用，标记「需要原生支持」
3. **预留接口**：UI 可创建/编辑/保存，但执行时跳过并记录「未实现」
4. **变通实现**：用 Flet 能力尽量模拟（如「进入应用」改成「打开应用对应 URL」）
