# 移动App工厂

> 让AI接管99%的移动应用开发工作，像扣子一样可编排、模型无关、可复用。

## 核心定位
- **形态**：扣子式工作流编排（节点化、参数化、可触发）
- **环境**：TRAE Work 云端沙箱执行（绕开本地硬件限制）
- **栈**：Python + Flet（Flutter引擎，一次编写全平台运行）
- **目标平台**：仅 Android
- **代码托管**：本地 + TRAE云沙箱（不用GitHub）

## 工作流7阶段
选题 → 计划 → 编程 → 调试 → 优化 → 封装 → 发布

每阶段对应：
- 一个工作流定义（`workflows/`）
- 一个或多个Skill节点（`skills/`）
- 确定性脚本（`scripts/`，模型无关）

## 任意模型可复用的实现原理
1. **骨架与执行分离**：工作流逻辑以文件形式存储（Skill定义 + Python脚本），模型只作为执行器
2. **提示词模板化**：每个Skill包含明确的输入/输出契约和提示词模板
3. **确定性逻辑脚本化**：构建、测试、打包等机械步骤用Python脚本固化
4. **记忆沉淀**：决策、经验、操作日志全部持久化到 `记忆/`
